# Idea Discovery

## Inputs

- user skills
- interests
- constraints
- available team
- timeline
- target outcome

## Process

1. Ask up to five clarifying questions only if needed.
2. Generate 3 to 5 directions across suitable tracks.
3. For each direction, state customer, pain, why now, first MVP, and proof needed.
4. Rank by feasibility, evidence potential, urgency, and distribution.
5. Recommend one direction and explain why.

## Output

- directions table
- recommended direction
- first validation task
- what not to build yet
- sources to check next
