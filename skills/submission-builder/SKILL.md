# Submission Builder

## Process

1. Identify the target format: pitch, grant, accelerator application, Devpost, Colosseum, ETHGlobal, or custom.
2. Map required fields.
3. Draft concise answers with proof, demo links, team strengths, and next milestones.
4. Flag missing assets.
5. Keep final submission under human control.

## Output

- completed draft
- missing fields
- asset checklist
- reviewer notes
- final-submit warning if relevant
