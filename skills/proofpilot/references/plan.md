# Plan

Use this workflow when the project needs a testable MVP, experiment, build path, timeline, or demo.

## Process

1. Define the user outcome and the assumption the MVP must test.
2. Choose the thinnest delivery model: manual service, concierge, spreadsheet, no-code workflow, prototype, web app, AI workflow, data notebook, web3 app, or physical pilot.
3. Separate must-have behavior from later features and explicit non-goals.
4. Fit the scope to team skill, time, budget, distribution, compliance, and program constraints.
5. Identify dependencies, credentials, external services, data, security work, and fallback paths.
6. Add instrumentation that can prove whether the MVP worked.
7. Produce a demo script that shows the promised outcome rather than a feature tour.

## Required Output

- MVP promise and target user
- assumption being tested
- delivery model and rationale
- must-have scope and non-goals
- milestone sequence with owner and acceptance condition
- cost, credentials, and dependencies
- risks and fallback paths
- measurement plan
- demo script

Do not recommend blockchain, AI, custom software, or automation unless it is necessary for the test or materially improves the chosen outcome.
